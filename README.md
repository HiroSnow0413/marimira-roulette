# マリミラルーレット (Marimira Roulette) - GPS対応版

スプレッドシート連動型の Web ルーレット。フロントエンドは GitHub Pages に静的配信し、バックエンド（項目取得・ログ保存）は Google Apps Script (GAS) で JSON API として公開する構成。当選時に GPS 緯度・経度を記録する不正防止機能付き。

## 🚀 主な機能
- **超高速読み込み**: CDN から静的ファイル配信＋軽量 JSON API なので GAS の「このアプリケーションは〜」警告を回避しつつ即表示。
- **高精度描画**: SVG を用いた放射状文字配置で、項目数が増えても文字が重ならない。
- **不正防止 (GPS)**: 保存時にスマホの緯度・経度を取得し、その場にいたことを記録。
- **快適な操作性 (Optimistic UI)**: 保存ボタンを押した瞬間に画面を閉じ、API への送信はバックグラウンド。
- **レスポンシブデザイン**: Bootstrap 5 採用。

## 📁 フォルダ・ファイル構成
- `index.html`: 静的エントリー HTML。Bootstrap と `src/` を読み込むだけ。
- `src/styles.css`: ページ固有の CSS。
- `src/script.js`: クライアント JS（SVG 描画、GPS 取得、API 叩き、UI 制御）。先頭の `API_URL` と `API_TOKEN` を環境に合わせて書き換える。
- `Code.gs`: GAS バックエンド。`doGet` で項目を JSON 返却、`doPost` でログ追記。スクリプトプロパティ `API_TOKEN` が必須。

## ⚙️ セットアップ

> 詳細な手順・スクリーンショット的な UI 指示・トラブルシューティングは **[SETUP.md](SETUP.md)** を参照。ここは要点のみ。

### 1. スプレッドシート
- シート「ルーレット」: A1 にタイトル、A2 以降に選択項目を記入。
- シート「ログ」: 自動生成されます（手動作成時は、日付, 時間, 項目, 名前, 緯度, 経度 の列を用意）。

### 2. GAS バックエンド
1. スプレッドシートから「拡張機能 → Apps Script」を開き、[Code.gs](Code.gs) の内容を貼り付け。
2. 左メニュー「プロジェクトの設定 → スクリプト プロパティ」で以下を追加：
   - キー: `API_TOKEN`
   - 値: 任意のランダム文字列（例: `openssl rand -hex 16` の出力）
3. 「デプロイ → 新しいデプロイ」でウェブアプリとして公開：
   - 実行ユーザー: **自分**
   - アクセスできるユーザー: **全員**
4. 表示された URL を控えておく。再デプロイするときは「デプロイを管理」から既存デプロイを更新すれば URL が維持される。

### 3. フロントエンド（GitHub Pages）
1. [src/script.js](src/script.js) 冒頭の 2 定数を書き換え：
   - `API_URL`: 手順 2-3 で控えた GAS の URL
   - `API_TOKEN`: GAS に設定したのと同じ文字列
2. このディレクトリを GitHub リポジトリにプッシュ。
3. リポジトリ設定 → Pages → Source = `main` branch / `root` に設定。
4. 数十秒後に `https://<username>.github.io/<repo>/` で公開される。

### 4. 位置情報の許可
- 初回スピン時にブラウザから位置情報の利用許可を求められる。許可するとログに緯度・経度が記録される。

## 🔐 認証トークンについて
`API_TOKEN` はフロント JS にもベタ書きされるため、公開リポジトリでは内容が見える。完全な秘匿はできない前提で、あくまで「bot が URL だけで書き込みを飛ばすのを防ぐ」程度の効果と理解する。気になるなら定期的にローテーション（GAS のスクリプトプロパティ＋フロント定数を同じ値に更新）する運用を取る。

## 📝 ログ記録フォーマット
「ログ」シートには、以下の形式でデータが蓄積される：

| 日付       | 時間     | 項目    | 名前     | 緯度    | 経度     |
| :--------- | :------- | :------ | :------- | :------ | :------- |
| 2026/04/18 | 10:00:00 | 当たりA | 山田太郎 | 35.6812 | 139.7671 |

## ⚠️ 注意事項
- **プライバシー**: 位置情報を取得するため、利用者にその旨を事前に周知することを推奨。
- **通信環境**: GPS の取得には数秒かかる場合あり（ルーレットの回転中に並行取得）。
- **HTTPS 環境**: Geolocation API は HTTPS 必須。GitHub Pages / GAS どちらも HTTPS なので問題なし。
